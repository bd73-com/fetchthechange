Add Browserless usage tracking and tier-based caps to prevent bill shock:

BACKGROUND:
We have a Browserless subscription: 1,000 sessions/month at $29/month. Currently at 376/1,000 (38%) usage. Need to track and cap usage per tier to prevent overages ($0.05/session after 1,000).

TIER CAPS:
- Free tier: 0 sessions (no Browserless access, static HTML only)
- Pro tier: 200 sessions/month per user
- Power tier: 500 sessions/month per user
- System-wide: 1,000 sessions/month (hard limit)

Add index on (userId, timestamp) for efficient monthly queries.

3. Create usage tracking service at server/services/browserlessTracker.ts:
```typescript
class BrowserlessUsageTracker {
  // Check if user can use Browserless this month
  async canUseBrowserless(userId: string): Promise<boolean>
  
  // Check system-wide cap
  async isSystemCapReached(): Promise<boolean>
  
  // Record a Browserless session
  async recordUsage(userId: string, monitorId: number, duration: number, success: boolean): Promise<void>
  
  // Get current month usage for a user
  async getUserMonthlyUsage(userId: string): Promise<number>
  
  // Get system-wide usage for current month
  async getSystemMonthlyUsage(): Promise<number>
  
  // Get top consumers
  async getTopConsumers(limit: number): Promise<Array<{userId: string, monitorId: number, monitorName: string, callCount: number}>>
}
```

4. Modify server/services/scraper.ts extractWithBrowserless() function:
- Before calling Browserless, check BrowserlessUsageTracker.canUseBrowserless(userId)
- If user cap exceeded OR system cap reached, skip Browserless and return null
- After successful Browserless call, record usage with BrowserlessUsageTracker.recordUsage()
- Log degradation to ErrorLogger when caps prevent Browserless usage
