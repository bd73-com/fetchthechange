Implement an internal error logging system for FetchTheChange to track scraping failures, email delivery issues, and API errors.

## Requirements

**1. Database Schema** (shared/schema.ts)
Add errorLogs table:
- id (serial, primary key)
- timestamp (timestamp, default now)
- level (text: 'error' | 'warning' | 'info')
- source (text: 'scraper' | 'email' | 'api' | 'scheduler' | 'stripe')
- errorType (text, nullable)
- message (text)
- stackTrace (text, nullable)
- context (jsonb, nullable) - sanitized, no passwords/tokens
- resolved (boolean, default false)
- resolvedAt (timestamp, nullable)
- resolvedBy (text, nullable)

**2. Logger Service** (server/services/logger.ts - NEW FILE)
Create ErrorLogger class with static methods:
- log(level, source, message, error?, context?)
- error(source, message, error?, context?)
- warning(source, message, context?)
- info(source, message, context?)

Sanitize context: remove password/token/apiKey/secret fields, truncate strings >1000 chars.
Write to both database and console.

**3. Integration Points**
Add ErrorLogger calls to:
- server/services/scraper.ts: Log checkMonitor failures, Browserless errors, selector discovery failures
- server/services/email.ts: Log sendNotificationEmail success/failure (audit trail)
- server/services/scheduler.ts: Log iteration failures and monitor check errors
- server/routes.ts: Enhance error handler to log all caught errors
- server/webhookHandlers.ts: Log Stripe webhook processing errors

**4. Admin Dashboard** (client/src/pages/AdminErrors.tsx - NEW FILE)
React component with:
- Filter by level (all/error/warning/info)
- Table: timestamp, level badge, source, errorType, message
- Icons: XCircle (error), AlertTriangle (warning), Info (info)
- Fetch from /api/admin/error-logs?level=X&limit=100

**5. API Endpoint** (server/routes.ts)
GET /api/admin/error-logs (authenticated)
Query params: level (optional), limit (default 100, max 500)
Return logs ordered by timestamp desc
