## Overview

The changelog page at `/changelog` is powered by `client/src/data/changelog.ts`, which is auto-generated by `script/sync-changelog.ts` from GitHub Releases. Manual entries for releases predating the automation go in `client/src/data/changelog-seed.ts`. The sync script merges seeds with releases, with releases taking precedence for matching versions. The generated file must never be edited by hand — it will be overwritten on the next sync.

## Workflow

1. Read `client/src/data/changelog.ts` (first 5 lines) — confirm the auto-generated header comment.
2. Read `client/src/data/changelog-seed.ts` — understand the seed entry format.
3. Read `script/sync-changelog.ts` — understand the merge logic and output format.
4. Determine the task:
   - **Sync from GitHub Releases**: Run `GITHUB_TOKEN=<token> npx tsx script/sync-changelog.ts`. The token needs `public_repo` scope (or `repo` for private repos). Without a token, the script works but is subject to lower rate limits.
   - **Add a pre-automation entry**: Add an entry to the `changelogSeed` array in `client/src/data/changelog-seed.ts` with `{ version, date, body }`. Then run the sync script to regenerate `changelog.ts`.
5. Verify the generated `changelog.ts` starts with the auto-generated header comment.
6. Run `npm run check && npm run test`.

## Hard constraints

- NEVER edit `client/src/data/changelog.ts` by hand — it is auto-generated by `script/sync-changelog.ts` and will be overwritten on the next sync
- NEVER add a seed entry for a version that already has a GitHub Release — the release takes precedence and the seed entry will be ignored
- NEVER run the sync script without `npx tsx script/sync-changelog.ts` — do not attempt to replicate its logic manually
- NEVER modify the output format of `changelog.ts` — the `ChangelogEntry` interface and array export are consumed by `client/src/pages/Changelog.tsx`
- NEVER skip the verification gate — run `npm run check && npm run test` before committing
